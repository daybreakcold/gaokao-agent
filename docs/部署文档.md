# 部署文档

本文档介绍如何把「高考志愿填报规划 Agent」网页版部署到服务器供他人访问。本地开发启动请参考 [启动文档](./启动文档.md)。

---

## 0. 部署前须知

- 服务基于 Python 标准库 `ThreadingHTTPServer`，适合**小规模 / 内部使用**；如需支撑高并发，建议前置 Nginx 反向代理。
- 回答采用 **HTTP chunked 流式输出**，反向代理必须关闭响应缓冲（见第 3 节 Nginx 配置），否则前端会等到回答全部生成才一次性显示。
- API Key 只在服务端使用，不会下发到浏览器；但**服务本身没有登录鉴权**，公网部署请自行加防护（Nginx Basic Auth、IP 白名单、或仅内网开放）。
- 服务默认只监听 `127.0.0.1`，需要对外提供服务时通过 `HOST` 环境变量修改（如 `HOST=0.0.0.0`）。

## 1. 准备工作

服务器需要：

- Python 3.8+（`python3 --version`）
- 能访问 `api.deepseek.com`（以及 DuckDuckGo / Tavily，用于联网搜索）
- 一个 DeepSeek API Key

上传代码到服务器（以 `/opt/gaokao-agent` 为例）：

```bash
# 方式一：scp
scp -r gaokao-agent/ user@server:/opt/gaokao-agent

# 方式二：git（如果代码在仓库中）
git clone <你的仓库地址> /opt/gaokao-agent
```

在部署目录创建 `.env`：

```bash
DEEPSEEK_API_KEY=sk-你的key
TAVILY_API_KEY=tvly-你的key   # 可选
```

安装依赖：

```bash
cd /opt/gaokao-agent
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## 2. 方式一：systemd 常驻服务（Linux 推荐）

创建 `/etc/systemd/system/gaokao-agent.service`：

```ini
[Unit]
Description=Gaokao Agent Web Service
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/gaokao-agent
EnvironmentFile=/opt/gaokao-agent/.env
# 配合 Nginx 时保持监听本机即可；直接对外则改为 HOST=0.0.0.0
Environment=HOST=127.0.0.1
Environment=PORT=8010
ExecStart=/opt/gaokao-agent/.venv/bin/python web_app.py
Restart=on-failure
RestartSec=3
# 建议用非 root 用户运行
User=www-data

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now gaokao-agent

# 查看状态和日志
systemctl status gaokao-agent
journalctl -u gaokao-agent -f
```

更新代码后重启：

```bash
sudo systemctl restart gaokao-agent
```

> 临时方案：不用 systemd 也可以 `nohup .venv/bin/python web_app.py > app.log 2>&1 &` 后台运行，但进程崩溃后不会自动拉起，仅适合临时演示。

## 3. Nginx 反向代理 + HTTPS

用 Nginx 对外提供 80/443 端口，转发到本机 8010。**关键点：必须关闭 `proxy_buffering`，否则流式输出失效。**

```nginx
server {
    listen 80;
    server_name gaokao.example.com;

    location / {
        proxy_pass http://127.0.0.1:8010;
        proxy_http_version 1.1;

        # ↓↓↓ 流式输出必需 ↓↓↓
        proxy_buffering off;
        proxy_cache off;
        # 模型生成 + 联网检索可能耗时较长
        proxy_read_timeout 300s;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

启用配置：

```bash
sudo nginx -t && sudo systemctl reload nginx
```

配置 HTTPS（推荐，使用 Let's Encrypt 免费证书）：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d gaokao.example.com
```

如需简单的访问控制，可在 `location /` 中加：

```nginx
        auth_basic "Restricted";
        auth_basic_user_file /etc/nginx/.htpasswd;   # htpasswd -c 创建
```

## 4. 方式二：Docker 部署

项目未附带 Dockerfile，可在项目根目录创建如下 `Dockerfile`：

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY web_app.py web_tools.py system_prompt.md ./

# 容器内必须监听 0.0.0.0，否则宿主机无法访问
ENV HOST=0.0.0.0 PORT=8010
EXPOSE 8010

CMD ["python", "web_app.py"]
```

构建与运行：

```bash
docker build -t gaokao-agent .

docker run -d --name gaokao-agent \
  --env-file .env \
  -p 8010:8010 \
  --restart unless-stopped \
  gaokao-agent

# 查看日志
docker logs -f gaokao-agent
```

对外服务同样建议前置第 3 节的 Nginx（把 `proxy_pass` 指向容器映射端口即可）。

## 5. 验证部署

```bash
# 首页应返回 200 和 HTML
curl -I http://127.0.0.1:8010/

# 浏览器访问，发一条消息确认流式输出正常（文字应逐段出现而不是最后一次性出现）
```

若流式输出异常（等很久后整段出现），优先检查 Nginx 是否遗漏 `proxy_buffering off`。

## 6. 运维注意事项

| 事项 | 说明 |
|------|------|
| 密钥安全 | `.env` 权限建议 `chmod 600`，不要提交到仓库；泄露后立即在 DeepSeek 平台重置 |
| 更新提示词 | 修改 `system_prompt.md` 后需重启服务生效（启动时读取一次） |
| 日志 | systemd 方式看 `journalctl -u gaokao-agent`；Docker 方式看 `docker logs` |
| 费用 | 每轮对话消耗 DeepSeek API 额度，公网开放前建议在 DeepSeek 平台设置用量上限 |
| 无状态 | 对话历史保存在浏览器侧，服务端无持久化数据，重启/迁移无需备份数据 |
| 端口防火墙 | 使用 Nginx 时，8010 端口不要对公网开放（保持 `HOST=127.0.0.1` 或安全组只放行 80/443） |
